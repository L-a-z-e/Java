생성자 - 인스턴스가 생성될 때 호출되는 인스턴스 초기화 메서드
1. 생성자의 이름은 클래스의 이름과 같아야한다.
2. 생성자는 return 값이 없다.

연산자 new가 인스턴스를 생성하는 것이지 생성자가 인스턴스를 생성하는 것은 아니다.

Card c = new Card();
1. 연산자 new에 의해 메모리(heap) 에 Card 클래스 인스턴스가 생성
2. Card() 생성자가 호출되어 수행
3. 연산자 new의 결과로 생성된 Card의 인스턴스 주소가 참조변수 c에 저장됨

기본생성자 (default constructor)
클래스에 생성자를 정의하지 않고도 인스턴스를 생성할 수 있었던 이유 -> 컴파일러가 기본 생성자를 제공하기 때문

없는 경우 컴파일러에서 자동적으로 클래스명(){} 으로 기본생성자를 추가하여 컴파일 함
하나라도 생성자가 있는 경우에는 자동 생성을 안해주므로 주의

생성자에서 다른 생성자 호출
- 생성자의 이름으로 클래스 이름 대신 this를 사용
- 한 생성자에서 다른생성자를 호출할 때는 반드시 첫 줄만 가능

this는 참조변수로 인스턴스 자신을 가리킨다. static 메서드는 인스턴스 변수를 사용할 수 없음 -> this 를 사용 할 수 없음
